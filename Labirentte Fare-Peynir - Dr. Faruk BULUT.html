Labirentte Fare-Peynir

Fare Peynir Sorusu ve Çözümü

Hepimiz birbirinden karışık yollar ve duvarlarla dolu, çözmesi zor ve bir o kadar da eğlenceli labirentleri hatırlarız. Bazılarında amaç labirentten dışarı çıkmak, bazılarında ise labirentin içinde bulunan hedefe ulaşmaktır.Hedefe ulaşmak için bir çok yol deneriz, bütün olasılıklara bakar, nereye gideceğimize karar veririz.Bazen de bunlardan en kısa yolu tercih ederiz.Ancak ilk bakışta kolay gibi görünen bu labirentlerden dışarı çıkmak beklendiği kadar kısa sürmeyebilir. Size bu konuda yardım edecek, çoğumuzun evinde bulunan ve günde yaklaşık 2-3 saat başında durduğumuz, bir makine tanıyoruz. Bilgisayar dediğimiz bu makineyi genelde bilimsel amaçlarla kullanmasak da, bilimsel amaçlarla kullanıdığı alanların bulunduğu da bir gerçektir. Olimpiyat bu alanlardan bir tanesidir.Biz de bilgisayar olimpiyatçıları olarak, size bir labirent sorusu ile olimpiyatta çözdüğümüz sorulara örnek göstermek istedik. Soruda bizden istenen labirentteki fareyi peynire en kısa yoldan ulaştırmamızdır.

Sorunun Kuralları Aşağıdaki gibidir:

1.Fare labirentin dışına çıkılamaz.

2.Fare duvarlardan geçilemez veya üstünden atlanamaz.

3.Fare 4 yöne hareket edebilir(kuzey, güney, doğu, batı).

4.Labirent sınırları:N satır,M sütun olmak üzere NxM<10000x10000 'dir.


Bu soruyu çözmenin bir çok yolu olsa da popüler olan 3 yol vardır.

ÇÖZÜM 1

Dinamit Yardımı ile Çözmek:

Bilgisayara duyulan ihtiyacı ortadan kaldırır.

ÇÖZÜM 2

Stack Veri Yapısı Kullanarak Çözmek:

 

FILO(First in Last Out) yani ilk giren son çıkar mantığı ile çalışır.Stack yığın anlamına   gelir ve üst üste konulmuş kutulara benzer.Kutu almak istediğimizde en üstten alır, koymak istediğimizde de en üste koyarız.

Bu veri yapısını kullanarak fare peynir sorusunu şu şekilde çözebiliriz: Yapmamız gereken bulunduğumuz yerden gidebileceğimiz alternatif yönleri(sağ,sol,yukarı,aşağı) kontrol edip(duvarlar, labirentten taşıp taşmadığımız vs.) Stack veri yapısına koymaktır.Ardından Stack'teki en üstte bulunan gidiş yolunu ele alıp farenin yeni yerinde gidebileceği yeni yerleri Stack'e tekrar koymaktır.


ÇÖZÜM 3

Queue Veri Yapısı Kullanarak Çözmek:

FIFO(First in First Out) yani ilk giren ilk çıkar mantığı ile çalışır.Queue kuyruk anlamına gelir.Yandaki şekilde görüldüğü gibi kuyruğa ilk giren ilk çıkar.Kuyruğa girmek isteyen en arkadan girer.

Queue veri yapısı bu soru için Stack veri yapısından daha hızlı çalışır ve sistem kaynaklarını daha az tüketir.Queue labirenti tarayarak yani fare alternatif dört yöne sürekli dallanarak gittiği için daha kısa sürede peynire ulaşır ve ulaştığı anda bütün alternatif yollar denendiği için farklı bir yoldan tekrar başlama gereksinimini ortadan kaldırır.Stack veri yapısını kullanışsız kılan peynire ilk ulaşıldığında, geride kalan yolların hepsini taranmadığı için başa dönüp alternatif yollardan ilerleyerek peynire ulaşma işini tekrarlamasıdır.

Şimdi C programlama dili ile, bize verilen labirent yapısındaki farenin, peynire gidebileceği en kısa yolu bulan kodu inceleyelim:

/*Okuma fonksiyonunda duvarlar -1, yollar 0 olarak matrise atılıyor*/

void Oku(void)

{

int i,j;

      oku=fopen("fare.in","r");

      yaz=fopen("fare.out","w");

      fscanf(oku,"%d %d",&N,&M);

      fscanf(oku,"%d %d",&fx,&fy);

      fscanf(oku,"%d %d",&px,&py);

      fx--;fy--;px--;py--;/*fx fy=farenin konumu px py=peynirin konumu*/

      for(i=0;i<N;i++)

      {

            for(j=0;j<M;j++)

            {

                  fscanf(oku,"%d",&lab[i][j]);

                  if(lab[i][j])

lab[i][j]=-1;

            }    

      }                      

}

/*Farenin hangi yönlere gidebileceğini hesaplayan fonksiyon.Bu fonksiyonda dört yönü kontrol edip (labirentten taşıp taşmadığımız ve duvarlar),ardından ekle() fonksiyonu ile bulduğumuz yönleri Queue veri yapısına ekliyoruz*/

void Kontrol(int x,int y)

{

      {

            if(lab[x+1][y]==0 || lab[x+1][y]>lab[x][y]+1)

{

                  lab[x+1][y]=lab[x][y]+1;

                  ekle(x+1,y);

            }

      }

      if(x-1>=0 && lab[x-1][y]!=-1)

      {

            if(lab[x-1][y]==0 || lab[x-1][y]>lab[x][y]+1)

            {

                  lab[x-1][y]=lab[x][y]+1;

                  ekle(x-1,y);

            }

      }

      if(y-1>=0 && lab[x][y-1]!=-1)

      {

            if(lab[x][y-1]==0 || lab[x][y-1]>lab[x][y]+1)

            {

                  lab[x][y-1]=lab[x][y]+1;

                  ekle(x,y-1);

            }

      }

      if(y+1<M && lab[x][y+1]!=-1)

      {

            if(lab[x][y+1]==0 || lab[x][y+1]>lab[x][y]+1)

            {

                  lab[x][y+1]=lab[x][y]+1;

                  ekle(x,y+1);

            }

      }

}
